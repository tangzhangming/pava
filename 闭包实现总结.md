# Pava 闭包功能实现总结

## 实现概述

根据《Pava 闭包设计白皮书》，已实现 Pava 语言在 JVM 平台上的闭包（Closure）功能。

## 实现内容

### 1. AST 扩展 (`src/ast.rs`)

- 添加 `Expr::Closure(ClosureExpr)` - 闭包表达式类型
- 添加 `Expr::ClosureCall(Box<Expr>, Vec<Expr>)` - 闭包调用表达式
- 添加 `CaptureVar` 结构体 - 捕获变量定义（包含名称和是否为引用捕获）
- 添加 `ClosureExpr` 结构体 - 闭包表达式定义（参数、返回类型、捕获变量、函数体）

### 2. Lexer 扩展 (`src/lexer.rs`)

- 添加 `Token::Use` 关键字，用于 `use` 子句

### 3. Parser 扩展 (`src/parser.rs`)

- 添加 `parse_closure()` 方法 - 解析闭包表达式
- 添加 `parse_capture_vars()` 方法 - 解析 use 子句中的捕获变量列表
- 添加 `parse_expr_stmt()` 方法 - 解析表达式语句（用于测试）
- 扩展 `parse_type()` 方法 - 支持类型关键字 Token（TypeInt, TypeInt32 等）
- 添加 3 个单元测试验证闭包解析功能

### 4. 运行时支持类

创建了 `pava-runtime` 项目，包含：

- **`pava.lang.Ref`** - 用于包装原始类型和对象引用的容器类
  - 支持 int, long, double, boolean, String 等类型的 getter/setter
  - 实现引用捕获，允许闭包内部修改外部变量

- **`pava.lang.Callable`** - 统一的闭包调用接口
  - 定义 `Object call(Object... args)` 方法
  - 所有 Pava 闭包都实现此接口

### 5. 代码生成 (`src/codegen.rs`)

- 添加 `emit_closure()` 方法 - 生成闭包表达式代码
- 添加 `emit_closure_call()` 方法 - 生成闭包调用代码
- 添加 `emit_ensure_ref_wrapped()` 方法 - 确保变量被包装在 Ref 中
- 添加 `emit_box_value()` 方法 - 基本类型装箱
- 添加常量池操作方法 - 支持动态添加类、方法、字段引用

## 语法示例

```php
// 值捕获 - 只读
$factor = 10;
$multiply = function(int $x, int $y) use ($factor) : int {
    return ($x + $y) * $factor;
};

// 引用捕获 - 可修改外部变量
$counter = 0;
$increment = function() use (&$counter) : int {
    $counter = $counter + 1;
    return $counter;
};

// 递归闭包 - 必须使用引用捕获
$fib = function(int $n) use (&$fib) : int {
    if ($n <= 1) {
        return $n;
    }
    return $fib($n - 1) + $fib($n - 2);
};

// 闭包调用
$result = $multiply(3, 4);  // 返回 70
```

## 捕获模式

| 语法 | 模式 | 说明 |
|------|------|------|
| `use ($var)` | 值捕获 | 只读，符合 JVM effectively final 约束 |
| `use (&$var)` | 引用捕获 | 可读写，使用 Ref 包装器实现 |

## 字节码生成策略

### 值捕获
- 映射为 JVM 原生 Lambda 捕获
- 变量在闭包内外均不可重新赋值

### 引用捕获
- 编译器将变量包装在 `pava.lang.Ref` 对象中
- Lambda 捕获 Ref 对象引用
- 通过 `getfield/putfield` 访问实际值

### 递归实现
1. 在栈上创建 `Ref` 对象
2. 生成 Lambda 捕获 Ref 对象
3. Lambda 内部通过 `getfield` 获取实际函数
4. Lambda 生成后回填到 `Ref.value`


测试覆盖：

## 注意事项

1. 当前代码生成实现为简化版本，完整实现需要使用 `invokedynamic` 和 `LambdaMetafactory`
2. 运行时类 `Ref` 和 `Callable` 需要编译后放入 classpath
3. 类型 `int` 被映射为 `Int64`（与 Pava 的类型系统一致）
